## 1. 원칙 — 두 언어의 동등성

이 앨범의 핵심 정체성은 **한국어와 일본어가 동등하게 존재**한다는 것이다(기획안 2-2). 사이트도 이를 그대로 구현한다.

- 한국어를 원본, 일본어를 번역으로 두지 **않는다.** 두 언어를 동급으로 취급한다.
- 콘텐츠는 **번역이 아니라 재해석.** 각 언어로 가장 자연스럽게 전달되는 표현을 쓴다(앨범의 일본어 트랙 철학과 동일).
- "여보여보"처럼 두 언어에서 다르게 작동하는 표현은 그 묘미를 양쪽 페이지에서 각각 살린다.

대상 언어: **한국어(ko) · 일본어(ja) · 영어(en)** 3종. (en 추가 구현됨 — 전 콘텐츠·UI 영어 번역, `/en` 라우팅, KO/JA/EN 3-way 전환기, hreflang·OG locale 확장. 인라인 분기는 `tri(locale, ko, ja, en)` 헬퍼 사용.)

## 2. 라우팅

`01-IA` 4절과 동일. 경로 접두사 방식.

```
/ko/...   한국어
/ja/...   일본어
```

- **기본 진입(`/`) 처리:** `Accept-Language`로 일본어 우선이면 `/ja`, 그 외 `/ko`로 분기. 자동 분기는 1회만 적용하고, 사용자가 토글로 바꾸면 그 선택을 우선(쿠키/localStorage 저장).
- **슬러그는 언어 무관 영문 고정**(`/ko/album/namsan-tower`, `/ja/album/namsan-tower`) → URL 매핑 단순화.
- **언어 토글**은 항상 *지금 보던 페이지의 반대 언어 버전*으로 이동(홈으로 튕기지 않음).

## 3. 콘텐츠 분리 모델

`02-content-spec`의 구조를 따른다.

- **본문(서사·트랙 소개 등):** 언어별 파일(`*.ko.md` / `*.ja.md`)로 분리. 각 언어 원고를 독립적으로 다듬는다.
- **구조 데이터(JSON):** 키별로 `{ "ko": "...", "ja": "..." }` 다국어 객체. 슬러그·날짜·링크 등 언어 무관 값은 단일.
- **UI 문자열(메뉴·버튼·라벨):** 메시지 사전(`messages/ko.json`, `messages/ja.json`)으로 분리.

---

## 4. 폴백 정책

일본어 콘텐츠는 단계적으로 채워지므로(펀딩 원고·기획안 JA 재활용 → 점진 확대), 누락 시 동작을 정한다.

| 상황 | 동작 |
|------|------|
| JA 본문 없음 | 해당 페이지를 **숨기지 않고**, 한국어 본문 + "일본어 번역 준비 중(日本語版は準備中です)" 안내 노출 |
| JSON 특정 키 JA 없음 | 한국어 값으로 폴백(빈 화면 금지) |
| UI 문자열 누락 | 키 그대로 노출 금지 → 빌드 시 누락 키 경고 |

- 폴백이 일어난 페이지는 검색엔진에 잘못된 언어로 인덱싱되지 않도록 `hreflang`·`lang` 속성을 정확히 둔다(`06` 참조).

---

## 5. 메타데이터·SEO 연동 (`06`과 공유)

- 각 페이지에 `<html lang="ko|ja">` 정확히 지정.
- `hreflang` 상호 링크: `ko` ↔ `ja` + `x-default`.
- OG/타이틀·디스크립션도 언어별로 분리.

---

## 6. 콘텐츠 운영 흐름

1. 한국어 원고 작성/갱신(`*.ko.md`)
2. 일본어 재해석본 작성(`*.ja.md`) — 사토유키에가 일본어 감수 가능(일본어 네이티브)
3. 빌드 → 누락 언어 경고 확인 → 배포

> 일본어 1차 콘텐츠는 `funding_plan_ja.md`, `album_plan_ja.md`에 이미 상당 부분 존재 → 이를 출발점으로 재활용한다.
