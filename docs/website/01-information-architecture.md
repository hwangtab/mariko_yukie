## 1. 사이트맵

```
/  (홈)  — 히어로 + 두 사람 + 아티스트 + 앨범 + 음악의 뿌리(스토리) + 갤러리 + 남산타워 트리비아 + 트랙리스트
├─ /artists                 아티스트
│  ├─ /artists/mariko        마리코
│  └─ /artists/yukie         사토유키에
├─ /album                   앨범 《남산타워》 (개요 + 포지셔닝 + 트랙리스트 + 사양)
│  └─ /album/[trackSlug]     트랙 상세 (예: /album/namsan-tower)
├─ /video                   뮤직비디오
├─ /gallery                 갤러리 (MV 촬영 스틸) — 1차 구현됨
├─ /lyrics                  가사 (전곡 인덱스)
│  └─ /lyrics/[trackSlug]    곡별 가사 (한·일)
├─ /live                    공연·소식 (발매 공연 + 투어 + News)
├─ /about                   프로젝트 소개 / 크레딧 / 연락
└─ /press                   (2차) 프레스 키트 — 보도용 약력·이미지 다운로드
```

> 구현 메모: 초기 2차였던 **사진 아카이브를 `/gallery`로 1차 편입**(MV 촬영 스틸 14컷, 헤더 내비 포함). 홈에는 리서치 기반 **「음악의 뿌리」 스토리 섹션**과 **남산타워 트리비아**를 신설. 헤더 6개 슬롯 유지를 위해 가사는 트랙/푸터 동선으로 둠. 각 경로는 다국어 접두사를 갖는다 → 아래 4절.

## 4. URL · 다국어 라우팅

**경로 접두사 방식** 채택 (검색엔진·공유에 명확).

```
/ko/...   한국어
/ja/...   일본어
/         → 기본 언어로 리다이렉트 또는 Accept-Language 기반 분기 (정책 04 참조)
```

예시:

| 콘텐츠 | KO | JA |
|--------|-----|-----|
| 홈 | `/ko` | `/ja` |
| 마리코 | `/ko/artists/mariko` | `/ja/artists/mariko` |
| 앨범 | `/ko/album` | `/ja/album` |
| 트랙(남산타워) | `/ko/album/namsan-tower` | `/ja/album/namsan-tower` |
| 가사(남산타워) | `/ko/lyrics/namsan-tower` | `/ja/lyrics/namsan-tower` |

- **슬러그는 언어 무관 영문 고정**(`namsan-tower`)으로 양언어 URL 대응을 단순화한다.
- 언어 전환 시 **현재 보던 페이지의 반대 언어 버전**으로 이동(홈으로 튕기지 않음).
- 트랙 슬러그 표준 목록은 `02-content-spec`의 트랙 테이블에서 단일 관리.

## 5. 페이지 간 동선 (주요 흐름)

```
[SNS/검색 유입]
   │
   ▼
홈 ──→ 뮤직비디오 ──→ 텀블벅(외부)
 │  └─→ 앨범 ──→ 트랙 상세 ──→ 가사
 │              └─→ 텀블벅 / 음원(외부)
 ├─→ 아티스트(마리코/유키에) ──→ 디스코그래피·외부 링크
 └─→ 공연·소식 ──→ 텀블벅 / 공연 안내
```

- 모든 본문 페이지 하단에 **상황별 CTA 블록**(펀딩 기간=텀블벅, 발매 후=음원/공연)을 공통 배치.
- 트랙 상세 ↔ 가사는 상호 링크.

## 6. 콘텐츠 ↔ 라우트 매핑 요약

| 라우트 | 콘텐츠 소스 | 비고 |
|--------|-------------|------|
| `/artists/*` | `02` 아티스트 스키마 + 기획안 PART 1 | 서사 + 이력표 |
| `/album`, `/album/[slug]` | `02` 트랙 스키마 + 기획안 2-3 / 펀딩원고 수록곡 | 트랙별 비하인드 |
| `/video` | 유튜브 임베드 ID + 컨셉 텍스트 | 링크 확정 필요 |
| `/lyrics/*` | `docs/lyrics/` 정리본 | 공개 범위 확정 필요 |
| `/live` | `02` 이벤트 스키마 | 공연장·날짜 미정 반영 |
| `/about` | 기획안 프로젝트 팀/크레딧 | 크레딧 공개 협의 중 항목 주의 |

## 7. 결정 / 메모

- 사진 아카이브·프레스 키트는 2차로 분리하되, IA에는 미리 자리를 잡아 URL을 예약한다.
- News를 독립 섹션으로 둘지 `/live`에 통합할지는 운영 강도 확정 후 결정(`00` Open Q). 현재는 `/live`에 통합.
